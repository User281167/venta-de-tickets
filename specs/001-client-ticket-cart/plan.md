# Implementation Plan: Client-Side Ticket Cart

**Branch**: `dev` | **Date**: 2026-07-14 | **Spec**: [spec.md](./spec.md)

**Input**: User description: "Build client side cart shop for tickets type, no login requirement"
**Clarifications**: User specified Next.js project structure, localStorage persistence, modular components with useCallback/memo, maxPerUser constraint, vitest tests, buy-flow redirects to login if unauthenticated.

## Summary

Build a client-side ticket cart system within the existing `ticket-purchase` feature. Refactor the in-memory `useCart` hook to persist to localStorage, add modular components (add, remove, clear, buy), enforce `maxPerUser` limits, and redirect unauthenticated users to login on "buy". Deliver with vitest unit tests covering cart logic, localStorage sync, and UI states.

## Technical Context

- **Language/Version**: TypeScript (strict) / Next.js 15 (App Router)
- **Primary Dependencies**: React 19, Chakra UI v3, TanStack Query v5, Zod, `@tabler/icons-react`
- **Storage**: localStorage for cart persistence; `CartContext` + reducer for reactive state
- **Testing**: Vitest + `@testing-library/react` + `jsdom`
- **Target Platform**: Modern web browsers (Chrome, Firefox, Safari, Edge)
- **Project Type**: Web application — frontend (Next.js) + backend (Express API)
- **Performance Goals**: Cart operations feel instant (< 50ms); single re-render per action via reducer
- **Constraints**: Cart must work fully offline after initial ticket types fetch; cart survives refresh via localStorage; no login required for cart operations
- **Scale/Scope**: Single event cart (multi-event is out of scope); < 20 ticket types per event; single browser session
