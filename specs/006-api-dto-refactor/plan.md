# Implementation Plan: API-DTO Refactor

**Branch**: `012-api-dto-refactor` | **Date**: 2026-07-10 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `specs/006-api-dto-refactor/spec.md`

## Summary

Refactor frontend-backend communication: remove dead API endpoint references, introduce shared Zod DTOs validated on both ends, standardize connection layer pattern, and add Vitest+RTL tests with mock data. Aligns with existing frontend feature structure (`app/` routes only, `features/<domain>/` for logic) and Chakra UI v3 theme.

**Language/Version**: TypeScript 5.x (Node.js Express + Next.js 16)

**Storage**: PostgreSQL via Prisma/Supabase (existing — no new migrations)

**Testing**: Vitest + @testing-library/react + jsdom. Colocated `__tests__/` dirs per feature.

**Target Platform**: Railway (backend), Vercel/Next.js (frontend)

**Project Type**: Web application (monorepo: Express API + Next.js frontend)

**Performance Goals**: No new performance constraints. DTO validation under 50ms per request. Tests complete under 30s.
